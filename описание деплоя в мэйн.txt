Рабочий процесс
Ветка	Назначение	Деплой
develop	Разработка, эксперименты	Нет
main	Стабильный продакшн	Dokploy
Как работать
# Переключиться на develop (локально)
git checkout develop

# Работаете, коммитите, пушите
git add -A
git commit -m "..."
git push

# Когда всё готово → GitHub → Pull Request develop → main
# После merge → Dokploy redeploy
Dokploy уже деплоит main. Хотите настроить авто-деплой (push в main → автоматический редеплой)? Для этого в настройках приложения в Dokploy укажите Watch Paths = / и Branch = main.
